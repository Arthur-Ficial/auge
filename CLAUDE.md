# auge - Project Instructions

> **apfel is the golden example.** All patterns, conventions, and guardrails from [apfel](https://github.com/Arthur-Ficial/apfel) apply here. When in doubt, check how apfel does it.

## The Golden Goal

auge has ONE purpose:

> **Expose Apple's on-device Vision framework as a usable, powerful UNIX tool
> for OCR, image classification, and object detection.**

### One mode: UNIX tool

```bash
auge --ocr screenshot.png              # extract text from image
auge --classify photo.jpg              # what's in this image?
auge --barcode product.jpg             # scan QR code or barcode
auge --faces group.jpg                 # detect faces, count, positions
auge --ocr scan.pdf                    # OCR a PDF page
cat image.png | auge --ocr             # pipe-friendly
auge --ocr screenshot.png | apfel "summarize this"  # pipe into LLM
```

- Image in, structured data out
- Pipe-friendly, composable with other tools
- Works with `jq`, `xargs`, shell scripts
- `--json` output for machine consumption
- Respects `NO_COLOR`, `--quiet`, stdin detection
- Correct exit codes

### No server mode (by design)

There is no standard OpenAI API endpoint for vision/OCR. auge is a pure CLI tool. No `--serve` flag. This is intentional - it does one thing well and composes via unix pipes.

### Non-negotiable principles:

- **100% on-device.** No cloud, no API keys, no network. Ever.
- **Honest about limitations.** Vision framework capabilities vary by macOS version. Say what works.
- **Clean code, clean logic.** No hacks. Proper error types.
- **Swift 6 strict concurrency.** No data races.
- **Zero dependencies.** Vision framework ships with macOS. No downloads.

## Architecture

```
CLI (image file/stdin) --> Analyzer.swift --> Vision framework (on-device)
                            VNRecognizeTextRequest    (OCR)
                            VNClassifyImageRequest    (classification)
                            VNDetectBarcodesRequest   (barcodes)
                            VNDetectFaceRectanglesRequest (faces)
```

- `AugeCore` library: pure Swift, no Vision dependency, unit-testable
- Main target: Vision framework integration
- Apple framework: `import Vision`
- Tests: `swift run auge-tests` (pure Swift runner, no XCTest needed - same pattern as apfel)
- No Xcode required - builds with Command Line Tools only
- Requires macOS 10.13+ (OCR requires 10.15+, classification requires 12+)

## Build & Test

```bash
make install                   # bump patch + build release + install to /usr/local/bin
make build                     # bump patch + build release
swift build                    # debug build
swift run auge-tests           # run pure Swift unit tests
```

**Version is in `.version` file** (single source of truth). Same auto-bump pattern as apfel.

**Always use `make install` for testing changes.**

## Handling GitHub Issues

Same process as apfel:

1. **Fetch** the full issue with `gh issue view <n> --repo Arthur-Ficial/auge --json body,comments,title,author,labels`
2. **Vet** - does it align with the golden goal? Can you reproduce it?
3. **Fix** if valid - TDD, keep changes minimal, run all tests
4. **Release** if code changed - `make install`, package asset, `gh release create`, update homebrew tap
5. **Close** with a friendly, short, truthful comment
